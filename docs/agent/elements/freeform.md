## What it is

Arbitrary vector geometry â€” paths made of move/line/curve/arc commands in EMU coordinates, plus optional `geometry_xml` raw OOXML and a fallback raster image. Onboarding produces these from PowerPoint freeform shapes.

## Position on agent creation

**The agent does not author freeforms from scratch in v1.** Generating valid `geometry_xml`, `style_xml`, or EMU-coordinate `PathCommand` lists is hostile to LLM output and unlikely to produce visually clean results.

Instead, ship a small **preset library** of named freeforms the agent picks from by intent:

## Preset library (v1)

| Preset name | Description | Adjustments |
|---|---|---|
| `"arrow_thick"` | Block arrow pointing right | direction, length, head ratio |
| `"arrow_curved"` | Curved arrow ribbon | direction, curvature |
| `"callout_speech"` | Speech bubble with tail | tail position |
| `"callout_thought"` | Thought-bubble cloud | tail position |
| `"banner"` | Ribbon banner | tail style |
| `"badge"` | Star/seal/sunburst | point count, fill |
| `"chevron_arrow"` | Right-chevron arrow (process step) | direction |
| `"check"` | Checkmark | weight |
| `"cross"` | X-mark | weight |
| `"plus"` | Plus sign | weight |
| `"line_double"` | Double parallel line | gap |
| `"bracket_curly"` | { or } bracket | direction |
| `"bracket_square"` | [ or ] bracket | direction |
| `"flowchart_decision"` | Diamond | â€” (also available as BridgeShape) |
| `"flowchart_terminator"` | Pill/stadium | â€” |
| `"flowchart_data"` | Parallelogram | skew |

Many of these are also achievable via `BridgeShape` with the right `geometry_preset`. The freeform versions exist for cases where the OOXML preset doesn't render cleanly or the user needs an adjustable parameter.

**Builder strategy:** for `geometry_preset`-equivalents, route to `BridgeShape` (simpler). Reserve `BridgeFreeform` for true freeforms that don't have a clean preset (curved arrow, ribbon banner).

## Edit-only / not for v1 creation

- `paths`, `geometry_xml`, `style_xml` (raw vectors â€” not exposed)
- Custom `PathCommand` lists (not exposed)
- `fallback` raster (auto-generated by the renderer when needed)
- Adding new presets â€” edit the preset library file, not via API

## Gotchas

- **Preset coordinates are normalized 0â€“1** in a unit box, then scaled to position. Each preset has its own normalized path.
- **EMU vs inches.** Internal `transform_emus` uses EMU (914400 EMU/inch). Builder converts.
- **Rendering fidelity.** Native renderer for freeform = SVG (Phase 2). For now, rendering goes through matplotlib via PathCommand interpretation. A few presets may need a fallback raster.
